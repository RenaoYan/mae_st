## Prostate MAE Pre-training

This repo supports leave-one-cohort-out MAE pre-training on the prostate 3D
patch zarr caches generated by:

```bash
/newdisk/individuals/renao/3DFM/Prostate/create_metadata.py
```

Available cohorts:

```text
UPenn_OTLS5 UPenn_OTLS4 DoD Oxford
```

2D pretraining mapping with `--use_2d_pretrain`:

```text
mae_vit_base_patch16  -> CONCH / Couch.bin
mae_vit_large_patch16 -> UNI / uni.bin
mae_vit_huge_patch14  -> H-optimus-1 / H-optimus-1.pth
```

Available patch cache configs:

```text
p128_s0    128 chunks, non-overlapping
p256_s0    256 chunks, non-overlapping subset from the overlapping source zarrs
p256_s128  256 chunks, overlapping
```

Example: hold out `UPenn_OTLS5` and train on the other three cohorts with
non-overlapping 128 chunks:

```bash
python run_pretrain.py \
    --data_source prostate \
    --leave_one_cohort UPenn_OTLS5 \
    --prostate_patch_config p128_s0 \
    --model mae_vit_base_patch16 \
    --batch_size 1 \
    --epochs 100 \
    --mask_ratio 0.75 \
    --norm_pix_loss \
    --use_2d_pretrain \
    --blr 1e-3 \
    --warmup_epochs 5 \
    --decoder_embed_dim 512 \
    --decoder_depth 4 \
    --num_workers 8 \
    --output_dir ./output_dir/prostate_loco_UPenn_OTLS5_p128_s0 \
    --log_dir ./output_dir/prostate_loco_UPenn_OTLS5_p128_s0
```

For 256 chunks, switch only the cache config:

```bash
--prostate_patch_config p256_s0 \
    --model mae_vit_base_patch16
```

or use all overlapping 256 chunks:

```bash
--prostate_patch_config p256_s128
```

In prostate mode, when the original MAE-ST defaults are unchanged, the training
script automatically sets:

```text
input_size = 128 or 256 according to --prostate_patch_config; 224 for mae_vit_huge_patch14
num_frames = input_size
t_patch_size = 16
pred_t_dim = num_frames
```

That means the model reconstructs the whole 3D chunk rather than only the
8-frame target used by the original video pre-training setup.
